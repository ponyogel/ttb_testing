Test Case 1 – Open Transfer Flow from Home

Test Case ID: MT_01
Test Description: Verify that user can start the transfer flow from Home screen.

Preconditions:
1. User is logged in to the mobile banking application.

Test Steps:
1. Open the mobile banking app.
2. From the Home screen, tap the Transfer entry point.

Expected Result:
1. App navigates to Select Transfer Methods screen, showing options:
    - Transfer to Own Account
    - Transfer to Other Account
    - Transfer to PromptPay (Citizen ID / Mobile Phone)

------------------------------------------------------------------------------------------------

Test Case 2 – Successful Transfer to Own Account

Test Case ID: MT_02
Test Description: Verify successful money transfer to user’s own account.

Preconditions:
1. User is logged in.
2. User has sufficient balance.

Test Steps:
1. From Home screen, start the transfer flow.
2. On Select Transfer Methods, choose Transfer to Own Account.
3. On Enter Details, select valid From and To accounts and input a valid Amount.
4. Tap Next/Continue to proceed.
5. On Review to Confirm Transaction, review transaction details and tap Confirm.

Expected Result:
1. On Enter Details, transaction data passes validation and user is navigated to the review screen.
2. On Review to Confirm Transaction, transaction data is validated successfully.
3. App navigates to Completed Screen and displays a message such as “Transaction is completed”, showing transaction reference details.

------------------------------------------------------------------------------------------------

Test Case 3 – Inline Validation Error on Own Account Transfer

Test Case ID: MT_03
Test Description: Verify inline error message when invalid data is entered for Own Account transfer.

Preconditions:
1. User is logged in.

Test Steps:
1. Start the transfer flow from Home.
2. Select Transfer to Own Account.
3. On Enter Details, select From and To accounts but enter an invalid Amount (e.g. zero, negative, or empty).
4. Tap Next/Continue.

Expected Result:
1. Validate transaction data on Enter Details fails.
2. User remains on the Enter Details screen.
3. An inline error message is displayed near the Amount field explaining the issue (e.g. “Please enter a valid amount”).
4. No navigation to the review screen and no transaction is performed.

------------------------------------------------------------------------------------------------

Test Case 4 – Backend Error on Own Account Transfer (Bottom Sheet Error)

Test Case ID: MT_04
Test Description: Verify bottom sheet error message when validation fails on the Review step for Own Account transfer.

Preconditions:
1. User is logged in.
2. System is configured to simulate a validation failure during confirmation (e.g. insufficient balance detected at confirmation time).

Test Steps:
1. Start the transfer flow from Home.
2. Select Transfer to Own Account.
3. On Enter Details, select valid From/To accounts and a valid Amount.
4. Tap Next/Continue to go to the review screen.
5. On Review to Confirm Transaction, tap Confirm.

Expected Result:
1. Enter Details validation passes and user reaches Review screen.
2. During Validate transaction data at the review step, validation fails.
3. A bottom sheet error message is displayed (e.g. “Transaction failed – insufficient balance / technical error”).
4. User remains on or is returned to an appropriate screen; transaction is not completed; no “Transaction is completed” screen is shown.
